# Source Evaluation Criteria

How to evaluate sources for inclusion.

---

## Criteria

### 1. Quality (1-5)

| Score | Description |
|-------|-------------|
| 5 | Exceptional — well-written, accurate, comprehensive |
| 4 | Good — minor issues only |
| 3 | Acceptable — some concerns but usable |
| 2 | Poor — significant issues |
| 1 | Unacceptable — major problems |

**Check:**
- Accuracy of information
- Writing quality
- Code example quality
- Up-to-date content

### 2. Relevance (1-5)

| Score | Description |
|-------|-------------|
| 5 | Core — directly about Claude Code |
| 4 | High — closely related |
| 3 | Medium — tangentially related |
| 2 | Low — loosely connected |
| 1 | None — not relevant |

**Check:**
- Directly applicable to Claude Code?
- Provides actionable patterns?
- Fits a fu level?

### 3. Stability (1-5)

| Score | Description |
|-------|-------------|
| 5 | Permanent — official, long-term |
| 4 | Stable — regular updates, maintained |
| 3 | Active — being maintained |
| 2 | Uncertain — sporadic updates |
| 1 | At risk — may disappear |

**Check:**
- Update frequency
- Maintenance status
- Platform stability

### 4. Uniqueness (1-5)

| Score | Description |
|-------|-------------|
| 5 | Essential — unique information |
| 4 | Valuable — useful additions |
| 3 | Helpful — nice to have |
| 2 | Redundant — mostly covered elsewhere |
| 1 | Duplicate — fully covered |

**Check:**
- What does this add?
- Is it covered by other sources?
- Unique perspective?

---

## Thresholds

| Score | Decision |
|-------|----------|
| 16-20 | Whitelist |
| 12-15 | Consider |
| 8-11 | Watch |
| 0-7 | Reject |

---

## Notes

- Quality and Relevance are weighted most heavily
- Low Uniqueness can be offset by high Quality
- Stability concerns can be mitigated by archiving
